<h1><strong>Supermarket Inventory Manager (PoC)</strong></h1>
<p><br />This is a full-stack reactive inventory management system built with React and Supabase. This project demonstrates real-time data updating, relational database design, and analytics.</p>
<h2>Features</h2>
<p><br />&bull; Real-Time Synch: Utilises PostgreSQL replication via Supabase Realtime to update the UI instantly across all clients without the page refreshing.<br />&bull; Multi-view Navigation: Full CRUD operations, inline editing, and simple live inventory analytics. (total value, out of stock, and low stock alerts).<br />&bull; Customer Storefront: Interactive ordering system that handles stock deductions and availability checks<br />&bull; Restock Portal: Admin interface for inventory ordering<br />&bull; Supplier Directory: Relational view mapping many-to-many relationships between vendors and catalogue items.<br />&bull; Relational Database Design: Products mapped to multiple suppliers via a junction table<br />&bull; Configured Row-Level Security policies to manage anonymous data access</p>
<h2>Tech Stack</h2>
<p><br />&bull; Frontend: React, Vite<br />&bull; Backend/Database: Supabase<br />&bull; State Management: React Hooks<br />&bull; API/Realtime: Supabase JS Client</p>
<h3>Database Schema</h3>
<p><br />To replicate this database, run the following in your Superbase SQL Editor:</p>
<blockquote>
<p>create table public.categories (<br /> id serial not null,<br /> name text not null,<br /> constraint categories_pkey primary key (id),<br /> constraint categories_name_key unique (name)<br />) TABLESPACE pg_default;</p>
<p><br />create table public.products (<br /> id serial not null,<br /> name text not null,<br /> category_id integer null,<br /> price numeric(10, 2) null default 0.00,<br /> quantity integer null default 0,<br /> constraint products_pkey primary key (id),<br /> constraint products_category_id_fkey foreign KEY (category_id) references categories (id) on delete set null<br />) TABLESPACE pg_default;</p>
<p>create table public.products (<br /> id serial not null,<br /> name text not null,<br /> category_id integer null,<br /> price numeric(10, 2) null default 0.00,<br /> quantity integer null default 0,<br /> constraint products_pkey primary key (id),<br /> constraint products_category_id_fkey foreign KEY (category_id) references categories (id) on delete set null<br />) TABLESPACE pg_default;</p>
<p>create table public.product_suppliers (<br /> product_id integer not null,<br /> supplier_id integer not null,<br /> constraint product_suppliers_pkey primary key (product_id, supplier_id),<br /> constraint product_suppliers_product_id_fkey foreign KEY (product_id) references products (id) on delete CASCADE,<br /> constraint product_suppliers_supplier_id_fkey foreign KEY (supplier_id) references suppliers (id) on delete CASCADE<br />) TABLESPACE pg_default;</p>
